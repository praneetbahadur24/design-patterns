# Builder Pattern

- Builder pattern is a type of creational pattern used to create complex object step by step.
- Builder collect data gradually.
- Vaidate the data.
- Create final object at one place.

# Why is the Builder Pattern Required?

- To solve telescopic constructor problem
- Improve readability
- Handles optional fields cleanly
- Centralize validation.
- Keep object immutable.

# When SHOULD you use Builder Pattern?

- Object has many field
- Many field are optional
- Construction logic is complex
- You want readable, fluent APIs.
