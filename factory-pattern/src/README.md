# Factory Pattern

- Factory pattern is creational design pattern that focus on object creation
- Instead of creating object directly with new you can delegate the creation logic to factory.

# Why factory pattern is required?

- Encapsulates object creation
- Loose coupling(Your code depend upon interface)
- Open/Close principle.
- mproves readability & maintenance

# Where SHOULD You use factory pattern?

- where you have multiple implementation
  - Payment Methods: Creditcard, UPI, Paypal
  - Notifications: SMS, Email
  - Database Connections: MYSQL, PostgresSQL, Mongodb
